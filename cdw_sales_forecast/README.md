# CDW Sales Forecast App

A Databricks-powered sales forecasting application for CDW Corp sales teams. Combines invoiced revenue, confirmed orders, partial revenue recognition, and Salesforce-style pipeline data into an interactive forecast that sales reps and managers can review and adjust.

## Architecture

```
Raw Data (UC Volume)
    │  Parquet files generated by scripts/generate_data.py
    ▼
Bronze Layer (Streaming Tables)
    │  AutoLoader ingestion from /Volumes/.../raw_data/
    ▼
Silver Layer (Materialized Views)
    │  Cleaned, typed, validated data
    ▼
Gold Layer (Materialized Views)
    │  Aggregated: monthly summaries, active deals, invoice details, category breakdowns
    ▼
Lakebase (Snapshot Sync)
    │  PostgreSQL tables synced from gold layer
    ▼
Streamlit App
    │  Dashboard, feedback, forecast builder, manager review, report generation
```

## Data Model

| Dataset | Description | Key Fields |
|---------|-------------|------------|
| **Sales Reps** | 35 CDW sales reps across 5 regions | rep_id, region, role, annual_quota |
| **Customers** | 800 customers across 8 verticals | customer_id, tier, vertical, assigned_rep_id |
| **Invoices** | 15,000 invoices with recognition status | invoice_amount, status, revenue_recognized_pct |
| **Orders** | 12,000 confirmed hardware orders | order_amount, ship_status, is_confirmed |
| **Deals** | 4,000 Salesforce-style pipeline deals | deal_amount, stage, probability, forecasted_amount |

## App Features

1. **Dashboard** — KPI cards (invoiced, confirmed, recognized, pipeline), trend charts, deal funnel, category breakdown
2. **Sales Feedback** — Reps adjust deal-level forecasts and add qualitative notes (risks, upside, confidence)
3. **Forecast Builder** — Blends actual invoiced revenue with weighted pipeline and rep adjustments
4. **Manager Review** — Review all rep submissions, approve/override team forecast
5. **Generate Report** — Download CSV reports for monthly summary, pipeline, invoices, or forecast vs actuals

## Deployment

### Prerequisites

- Databricks CLI installed and `group-demo` profile configured
- Access to a SQL Warehouse (referenced in databricks.yml)

### Step 1: Deploy the bundle

```bash
cd cdw_sales_forecast

# Validate
databricks bundle validate -t dev

# Deploy pipeline, volume, and app
databricks bundle deploy -t dev
```

### Step 2: Generate synthetic data

Run the data generation script on a Databricks cluster:

```bash
# Upload and execute
databricks fs cp scripts/generate_data.py dbfs:/tmp/generate_data.py --profile group-demo
```

Or use MCP tools / a notebook to run `scripts/generate_data.py` on a cluster.

### Step 3: Run the pipeline

```bash
databricks bundle run cdw_sales_pipeline -t dev
```

### Step 4: Set up Lakebase sync

Run `scripts/setup_lakebase_sync.py` on a cluster to create the Lakebase instance and sync gold tables.

### Step 5: Start the app

```bash
# Add Lakebase resource to the app in the Databricks UI
# Then start the app
databricks bundle run cdw_sales_app -t dev
```

### Step 6: Check logs

```bash
databricks apps logs cdw-sales-forecast-dev --profile group-demo
```

## Local Development

Set `USE_MOCK_BACKEND=true` to run the app locally with in-memory mock data:

```bash
cd src/app
USE_MOCK_BACKEND=true streamlit run app.py
```
